# Vehicle catalog data

The catalog deliberately uses only sources whose provenance can be audited.

## `makes-carby-v1.json`

The make list, featured flags and required aliases are copied from the project requirements dated
2026-07-19. It contains no listing counts. `Эксклюзив` is marked as a special category.

## `catalog-wikidata-2026-07-19.json`

Generated by `pnpm catalog:fetch -- --version=2026-07-19` from the official Wikidata Query Service.
Wikidata structured data is published under
[Creative Commons CC0 1.0](https://www.wikidata.org/wiki/Wikidata:Licensing).

The extraction rules are intentionally conservative:

- a model must be explicitly typed as `automobile model series (Q59773381)`;
- a generation must be explicitly typed as `car model (Q3231690)` and linked to a series through
  `part of the series (P179)`;
- production years come only from explicit inception/start and discontinued/end statements;
- model codes come only from `model number (P13351)`;
- source records that normalize to the same make/model identity are merged deterministically and
  their generations are re-linked to one canonical model;
- unknown years, codes and relationships remain `null`;
- no records are scraped from commercial automotive sites.

The generated file contains an `extractionReport` with every skipped model/generation and known
coverage limitations. It is a useful, legally reusable baseline, not a claim of a complete global
automobile catalog.

See [`COVERAGE.md`](./COVERAGE.md) for the summarized counts and the complete list of required makes
for which this conservative dataset has no verified model series.
